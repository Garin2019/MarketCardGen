# Marketplace Card App

Приложение для подготовки карточки товара на основе фото товара и пользовательских параметров. Оно помогает пройти полный сценарий от загрузки исходного изображения до получения готового комплекта материалов: текстового описания, набора товарных изображений, поста для медиа и итоговой таблицы для выгрузки.

## Что делает приложение

Основной сценарий работы:

1. Пользователь загружает фото и уточняет параметры генерации.
2. Приложение генерирует краткое и подробное описание товара на основе фото-референса (можно вносить свои правки).
3. На основе фото-референса и шаблонов создаются новые изображения для карточки товара.
4. Пользователь подготавливает публикацию (редактируемый текст) для медиа.
5. На финальном шаге получает итоговую карточку товара, таблицу и изображения.

Ключевые возможности:

- генерация текстового описания товара по фото и параметрам;
- настроенные коннекторы к API: OpenAI, MiniMax, Qwen, Openrouter;
- настройка категории, артикула, целевой аудитории, тональности, температуры ИИ, ключевых слов и дополнительных требований;
- генерация изображений на основе фото товара или отдельного референса;
- использование шаблонов промтов для генерации изображений;
- встроенный модуль `editor` для отдельного редактирования изображений;
- генерация и публикация постов в ВКонтакте, Telegram и MAX;
- экспорт итоговой карточки товара в `Excel` и `CSV`;
- выгрузка всех сгенерированных изображений одним `ZIP`-архивом.

## Архитектура приложения

Проект состоит из двух основных частей и одного встроенного модуля:

- `backend/` - API и бизнес-логика на FastAPI
- `frontend/` - пользовательский интерфейс на React
- `editor/` - отдельный frontend-модуль редактора изображений, встроенный в основной frontend
-  СУБД - SQLite 


### Editor

Папка `editor/` содержит отдельное React-приложение редактора изображений. Сейчас оно подключено как независимый frontend-модуль и открывается из главного меню основного приложения. Оно не сохраняет изменения обратно в backend автоматически, но доступно пользователю как отдельный инструмент.


### Хранение данных

Локальные данные сохраняются в папке `data/`:
- `data/app.db` - база данных `SQLite`;
- `data/uploads/` - загруженные и сгенерированные изображения;
- `data/logs/` - файлы логов приложения.


## Запуск проекта

### 1. Что нужно установить

Перед запуском убедитесь, что на компьютере установлен Docker.
Для Windows и macOS:
- установите `Docker Desktop`
- ссылка: [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)

Для Linux:
- установите `Docker Engine`
- установите `Docker Compose Plugin`


### 2. Подготовьте переменные окружения

В корне проекта создайте файл `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```
Или просто вручную переименуйте `.env.example` в `.env`.

Откройте файл `.env` и заполните данные провайдера ИИ (ключи API) и параметры доступа к медиа:

- `OPENROUTER_API_KEY`
- `DASHSCOPE_API_KEY`
- `OPENAI_API_KEY`
- `MINIMAX_API_KEY`
- `VK_ACCESS_TOKEN`
- `VK_OWNER_ID`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHANNEL_ID`
- `MAX_ACCESS_TOKEN`
- `MAX_CHANNEL_ID`

### 3. Запустите контейнеры

Убедитесь, что Docker запущен!

Из корня проекта выполните:
```powershell
docker compose up --build
```

После первого запуска будут собраны два контейнера:
- `cardgen-backend`
- `cardgen-frontend`

Для Windows также доступны готовые bat-файлы (вместо команд powershell):
- `install-docker-app.bat` - первичная сборка образов через `docker compose build`
- `start-docker-app.bat` - запуск контейнеров через `docker compose up`
- `stop-docker-app.bat` - остановка контейнеров через `docker compose down`


### 4. Адреса для открытия приложения в браузере (после успешного запуска контейнера)

- frontend: [http://localhost:3000](http://localhost:3000)
- editor: [http://localhost:3000/editor/](http://localhost:3000/editor/)



## Примечания

- Данные сохраняются локально в папке `data/`.
- При первом старте backend автоматически создаёт таблицы и системные шаблоны.
- Системные шаблоны промтов синхронизируются при запуске backend.
- Публикация изображений в отдельных соцсетях зависит от ограничений конкретного API (проблемы с VK, где доступ на публикаию картинок в сообщество по запросу) и типа токена.
- Функционального тестирование API Qwen и Minimax не проводилось.
